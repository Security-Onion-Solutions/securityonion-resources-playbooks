# Security Onion Playbooks

## Directory Structure

- `playbooks-raw/` — Raw templated playbooks (generated by AI/automation)
- `public/` — Expanded playbooks (processed from templates, ready for validation)
  - `sigma/` — Playbooks for Sigma detection rules
  - `nids/` — Playbooks for NIDS rules
  - `yara/` — Playbooks for YARA rules
- `securityonion-normalized/` — Normalized output (auto-generated, do not edit by hand)
- `templates/` — Template definitions
  - `working_templates.yaml` — Query templates for playbook generation
- `scripts/` — Processing scripts
  - `batch_template_processor.py` — Expands templated playbooks
  - `validate_playbooks.py` — Validates expanded playbooks
  - `normalize_playbooks.py` — Creates normalized versions
  - `patterns_sigma.yaml` — Substitution patterns for Sigma rules and engine
  - `patterns_nids.yaml` — Substitution patterns for NIDS rules and engine

## Playbook ID Numbering Scheme

Playbook IDs use a 7-digit numbering system:
- **Range**: 1,000,000 - 1,999,999 (reserved for Security Onion Playbooks)
- **Starting Point**: 1,200,001
- **Format**: 7-digit number (e.g., 1200001, 1200002, 1200003)
- **Assignment**: Start at 1,200,001 and increment sequentially

## Playbook Processing Workflow

### Complete Workflow
1. **Generate raw playbooks** → Save to `playbooks-raw/`
2. **Expand templates** → Process with template processor
3. **Validate** → Check and fix expanded playbooks
4. **Normalize** → Create final versions


### Step 1: Generate Raw Playbooks
This is done by hand, or by an AI/automation process.

### Step 2: Expand Templates
```bash
python3 scripts/batch_template_processor.py \
  --input-dir ./playbooks-raw/nids/individual \
  --output-dir ./public/nids/individual \
  --template-file ./templates/working_templates.yaml
```

### Step 3: Validate Expanded Playbooks
This will validate and fix any issues with the expanded playbooks.
```bash
python3 scripts/validate_playbooks.py -f ./public/nids/individual
```

### Step 4: Normalize Playbooks
This will normalize the playbooks for Security Onion.
#### Normalize All Files (One-Time/Full Run)
```bash
python3 scripts/normalize_playbooks.py --all
```
- Normalizes **all** YAML files in `public/` and writes to `securityonion-normalized/`.
- Deletes any orphaned files in `securityonion-normalized/`.

#### Normalize Only Staged Files (Pre-commit Mode)
```bash
python3 scripts/normalize_playbooks.py --staged
```
- Only normalizes files staged for commit (added/modified/deleted).
- Intended for use with pre-commit hooks.

## Setting Up Pre-commit Hooks

1. Install pre-commit (if not already):
   ```bash
   pip install pre-commit
   ```
2. Add a `.pre-commit-config.yaml`
3. Install the hook:
   ```bash
   pre-commit install
   ```

## Contribution Workflow

### For Manual Edits
- Edit files directly in `public/`
- Never edit `securityonion-normalized/` by hand
- Use the normalization script to keep everything in sync

### For Generated Playbooks
1. Generate templated playbooks and save to `playbooks-raw/`
2. Run the template processor to expand them into `public/`
3. Validate the expanded playbooks
4. Run normalization to create final versions

### Important Notes
- Raw playbooks in `playbooks-raw/` use template references for queries
- The template processor expands these into full Sigma queries
- Always validate expanded playbooks before normalization
- The normalized versions are what get deployed to Security Onion
